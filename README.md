# 人脸识别系统 (hw03)

基于 `face_recognition` 和 `Streamlit` 的人脸识别应用，实现了人脸检测、识别和可视化展示功能。

## 项目结构

```
hw03/
├── app.py                  # Streamlit 前端主入口
├── requirements.txt        # 依赖包列表
├── README.md               # 项目说明文档
│
├── src/                    # 源代码目录
│   ├── __init__.py         # 空文件，让 src 成为一个 Python 包
│   └── face_processor.py   # 核心人脸处理算法
│
├── known_faces/            # 存放已知人脸图片，如 "lhc1.png"
└── examples/               # 存放供用户测试的示例图片，如 "lhc3.png", "lhc4.png"
```

## 功能说明

- **人脸检测**：使用 `face_recognition` 库检测图片中的人脸位置
- **人脸识别**：提取人脸的 128 维特征编码，并与已知人脸库进行比对
- **Web 界面**：使用 Streamlit 构建交互式 Web 界面
- **支持多人脸**：可以同时检测和识别多张人脸
- **阈值调节**：可调节识别阈值，控制识别严格程度
- **置信度显示**：显示人脸识别的置信度百分比
- **位置信息**：显示检测到的人脸位置坐标
- **实时反馈**：处理过程中显示加载动画，提升用户体验
- **错误处理**：完善的错误捕获和提示机制
- **响应式设计**：适配不同屏幕尺寸

## 如何使用

### 1. 准备人脸库
在 `known_faces/` 目录下放置已知人脸图片：
- 图片格式：jpg、png、jpeg
- 文件名：使用英文或拼音命名（如 "lhc1.png"）
- 每张图片应只包含一张清晰的人脸

### 2. 准备示例图片
在 `examples/` 目录下放置供测试的示例图片：
- 可以是单人照或多人合照
- 图片格式：jpg、png、jpeg

### 3. 运行应用

```bash
streamlit run app.py
```

启动后，浏览器访问显示的本地 URL（默认 http://localhost:8501）

## 技术原理

1. **人脸检测**：使用 HOG (方向梯度直方图) 模型检测人脸位置
2. **特征提取**：将人脸转换为 128 维的特征向量
3. **人脸识别**：计算欧氏距离，与已知人脸库进行比对
4. **结果展示**：在图片上绘制人脸框和识别结果
5. **模块分离**：核心算法与前端界面分离，提高代码可维护性
6. **EXIF 处理**：自动处理图片的 EXIF 信息，防止手机照片侧翻

## 注意事项

- `face_recognition` 库依赖 `dlib`，安装过程可能需要一些时间
- 首次运行时会加载人脸库，可能需要几秒钟时间
- 识别精度取决于人脸库中图片的质量和数量
- 对于多人脸图片，识别速度会稍微变慢
- 确保图片光线充足，人脸清晰，以获得最佳识别效果
- 对于分辨率过高的图片，处理速度可能会较慢

## 常见问题

### Q: 安装 dlib 失败怎么办？
A: 确保已安装 C++ 编译环境，然后尝试使用预编译的 dlib 版本：
```bash
pip install dlib==19.22.99
```

### Q: 识别结果不准确怎么办？
A: 可以尝试以下方法：
- 调整识别阈值
- 提高人脸库中图片的质量
- 确保人脸清晰可见，光线充足
- 确保人脸库中包含足够多的样本

### Q: 应用运行缓慢怎么办？
A: 对于多人脸图片，识别速度会较慢，这是正常现象。可以尝试使用分辨率较低的图片。

### Q: 上传图片失败怎么办？
A: 确保上传的图片格式正确（jpg、jpeg、png），且文件大小适中。

### Q: 人脸框位置不准确怎么办？
A: 确保图片光线充足，人脸清晰可见，且人脸在图片中占比适当。
